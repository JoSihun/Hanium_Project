(Temp에 있는 장고 가상환경 env는 윈도우용이므로 사용x)

3. 다양한 문제점들
가정 : runserver를 실행중인 계정과, html페이지를 추가/수정 하는 '타'계정이 각각 존재한다고 가정한다.

현상 1 : '타'계정이 html을 추가/수정해도 runserver를 돌리는 계정은 따로 있기 때문에 '타'계정은 추가/수정된 html 페이지를 확인할 수 없다.
-> 다른포트 (ex 8080) 등을 다른포트로 해결가능하나 추후 코드를 합치는 과정에서 merge 문제가 발생할 수 있음

현상 2 : '타'계정이 commit & push를 해도 runserver 계정이 pull 을 하지 않았기 때문에 '타'계정은 아직 추가/수정된 html 페이지를 확인할 수 없다.

현상 3 : runserver 계정이 pull을 했을 때 비로소 '타'계정은 추가/수정된 html 페이지를 확인할 수 있다.

현상 4 : 위의 모든 과정중 언제든지 merge가 발생할 수 있다. 이때 수정을 잘 못하면 commit & push가 정상적으로 작동하지 않을 수 있다.

현상 5 : commit & push를 하는과정에서 .idea 내의 수정사항은 commit & push하지 않아도 된다.

현상 6 : 수정과정에서 파이참의 Project 경로 내 Directory가 사라지는 현상 발생 (아직 미해결, 이때 다시 Open 해주어야한다. 이유는 모름)

현상 7 : 다른사람이 .html파일을 만들고 commit& push 한 상황에서 본인이 pull을 땡기고
똑같이 서버를 열어서 실행하면 다른사람이 등록한 .html파일은 서버에 없기때문에 인식못하는 상황이 발생, html파일 뿐만아니라 모든파일에도 적용됨
위와 같은 문제가 발생하는 이유는 리눅스 서버에 업로드가 되기위해서는 해당 파일이 수정되어야 하기 때문임..
즉, 파일을 다운로드 받아서 경로에 존재해도 그 파일을 건들지않으면 업로드가 되지않음
-> 파이참으로 git pull하면 리눅스 서버에서도 git pull을 해야 해결될거라고 예측중..
