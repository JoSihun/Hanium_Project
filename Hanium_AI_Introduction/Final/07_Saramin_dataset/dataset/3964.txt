코오롱그룹
전산전공
[도전하는 삶]
계속하여 노력하면 안 될 일은 없다고 생각하며 항상 도전하며 살아왔습니다. IT교과목을 수강하며 어려운 개념과 코드작성에 애를 먹었습니다. 그때마다 남아서 독학하고 관련 코드를 통째로 외우며 학습하였습니다. 어느 정도 IT학습에 관한 틀이 잡힌 후, 학교에서 배울 수 있는 것 외에 새로운 것을 배워 남들과 차별성을 주고자 하였습니다. 그리하여 빅 데이터 학습에 도전하였습니다.
빅 데이터 교육 과정 수강 시, 빅 데이터 개념과 관련 툴의 원리가 이해 안 돼 매우 힘들었습니다. 연구소에서 인턴 근무할 때에는 Spark라는 새로운 빅 데이터 분석 툴을 익히느라 매우 힘들었습니다. 하지만 계속 노력하면 안 될 일은 없다고 판단하여 야근, 나머지 공부를 자청하여 부족한 부분을 메꿔나갔습니다.
또 빅 데이터 관련 세미나와 학회에 참석하여 배우는 등 시간, 장소를 가리지 않고 빅 데이터를 이해하기 위해 노력하였습니다. 그 결과 빅 데이터 교육 과정도 수료할 수 있었고, 인턴 업무도 성공적으로 끝마칠 수 있었습니다. 앞으로도 코오롱 베니트에서 이러한 끊임없는 도전 정신을 바탕으로 시스템 개발 및 운영 업무를 수행하겠습니다.
[BA팀에 가고 싶습니다]
미래에 코오롱 베니트 BA팀에서 패션 유통이 아닌 다른 부분에 빅 데이터를 활용한 서비스를 만들고 싶습니다. 2014년 빅 데이터 학회에서 코오롱 fnc의 발표를 들으며 코오롱이 국내에서 빅 데이터를 적극 활용하는 기업임을 알게 되었습니다. 또 2015년 코오롱 베니트 kick-off meeting에서 Big Analytics팀을 만들었다는 소식을 들었습니다. 이렇게 적극적으로 빅 데이터를 활용하는 코오롱에서 제 빅 데이터 역량을 펼치고 싶습니다.
빅 데이터 관련 역량을 쌓기 위해 아래의 2가지를 경험하였습니다.
1. OO도청 D-school Big-data과정을 수강하여 빅 데이터 개념, Hadoop이나 SAP HANA에서 빅 데이터 처리하는 방법 등등 다양한 것을 배웠습니다.
2. 연구소에서 Nielsen시청률 데이터 분석 및 예측 프로젝트에 참가하여 Spark의 분석 원리와 Spark코드를 직접 작성해보았습니다. 앞으로 BA팀에서 시스템 개발 사양서에 따라 다양한 언어와 빅 데이터 분석 툴을 이용하여 여러 서비스를 개발하고 싶습니다.
[자신에 대한 믿음으로 프로젝트 성공적으로 이끌다]
한이음 프로젝트에 참여하여 SNS서비스를 기획하여 멘토님과 교수님께 긍정적인 반응을 끌어냈습니다. 2014년 한이음 프로젝트에 조장으로 참가하여 SNS서비스를 기획하였습니다. 주변에서 모두 나올만한 SNS서비스는 다 나와 있다며 부정적으로 보았습니다. 하지만 저는 타겟팅을 통하여 대외활동과 관련된 대학생들을 위한 SNS를 만들면 승산이 있을 것으로 생각했습니다. 하지만 여전히 주변인들은 취업시장보다 대외활동시장이 작아 수익성이 낮을 것이라 하였습니다. 이 문제 있어 저는 앞으로 취업 시장에 화두는 경험이라고 생각하여 제 생각을 믿고 추진하였습니다. 먼저 팀원들을 설득하고 주변의 불신을 이겨내기 위해 3가지 전략을 내세웠습니다.
첫째, 사전조사를 철저히 하고, 비즈니스 캔버스 모델을 바탕으로 기획한다.
둘째, 팀 내 옵저버를 두어 객관적으로 프로세스를 평가하게 한다.
셋째, 설문조사를 실행하여 외부 의견을 적극적으로 수용하도록 한다. 팀원들과 소통하고 위 3가지를 적극적으로 실행한 끝에 대외활동 관련 기업, 학생들을 위한 SNS서비스 기획안을 완성하였습니다. 기획안 발표를 한 후, 멘토님과 교수님께 좋은 반응을 받았습니다. 그리하여 성공적으로 프로젝트를 끝마쳤습니다. 실제로 프로젝트 기획도 해볼 수 있었고, 리더로서 팀원들과 소통하며 설계 커뮤니케이션 능력을 신장했던 좋은 경험이었습니다.
[긍정적 믿음의 결과]
인턴 근무를 할 때, 원본 데이터를 삭제한 적이 있었는데, 긍정마인드를 가지고 여러 방법을 시도하여 문제를 해결한 경험이 있습니다. 인턴 근무를 하는데 하루는 원본 데이터를 백업해야 했습니다. 백업 작업 중, 실수로 파일 경로에서 “/”를 빼서 원 데이터를 덮어씌워 버렸습니다. 선임연구원님은 미리 샘플링 한 데이터가 남아 괜찮다고 하셨지만, 나중에 결과를 도출할 때 필요하였습니다.
저는 분명 해결할 수 있을 것이란 믿음을 가지고 여러 가지 방법을 시도하였습니다. 친구, 교수님, 인터넷 커뮤니티 등등 백방으로 자문하고 인터넷에서 이러저러한 방법을 찾아 계속해서 시도하였습니다. 결국, 원하는 해결책을 못 찾고 포기하던 찰나 마지막으로 한 번 더 시도해보자는 생각으로 다른 서버들을 검색해보았습니다. 밤새 검색해보니 어느 한 서버에 원 데이터가 정제되어 나뉘어 있는 것을 발견했습니다.
바로 저는 데이터 변환 과정을 역으로 분석하고 JOIN 하여 결국 복구하였습니다. 포기할 수도 있었지만 ‘할 수 있다’라는 믿음을 가지고 여러 방법을 시도하여 문제를 해결한 좋은 경험이었습니다. 앞으로 코오롱 베니트에서 긍정적 믿음을 바탕으로 클라이언트의 요구와 타 시스템과의 연계 문제, 기능장애로 인한 기존 시스템 변경 문제, 업그레이드 및 기능 추가 문제 등 여러 문제를 해결하겠습니다.
[코오롱과 함께하는 미래]
단기적으로는 SNS데이터와 모바일기기로부터 개인 데이터를 얻어 그 사람의 감성을 분석하는 서비스를 내놓겠습니다. 사람의 기분을 분석하고 그때그때 코오롱의 재화나 서비스를 맞춤으로 제공하여 고객이 코오롱과 평생 함께하고, 행복한 기분이 들도록 만들겠습니다. 중기적으로는 모두에게 인정받는 빅 데이터 분석 전문가로 거듭나겠습니다. 특히 빅 데이터 분석 및 예측에 있어 뇌과학 분야와 접목을 통하여 데이터를 스마트하게 분석하는 전문가가 되겠습니다.
장기적으로 제 빅 데이터 역량을 바탕으로 코오롱의 지속적 성장에 이바지하겠습니다. 그리고 더 나아가 높은 수익을 내어 투자하고 싶은 코오롱을 만들겠습니다. 화학, 소재, 바이오, 건설, 레저 등등 다양한 분야에 데이터 분석 및 예측을 통한 통찰력을 제공하여 전 부분에 있어 성장세를 기록하게 하고 싶습니다. 또 해당 분야별 생태계 조성을 통해 하위 산업 육성은 물론 해당 산업의 리더가 될 수 있도록 만들겠습니다.
질문 : 빅 데이터 분석 및 예측 업무를 해봤다는데 더 자세하게 얘기해보세요.
답변 : 차세대 융합 기술 연구원 공공데이터 센터에서 인턴 생활을 하며 참여한 프로젝트는 기존의 Hadoop기반의 Nielsen시청률 데이터 분석 코드를 Spark기반으로 바꾸는 것이었습니다. Nielsen시청률 데이터가 계속해서 쌓이며 용량이 커지는데 Hadoop만으로는 분산 처리가 힘들고 시간이 오래 걸리니 인메모리 방식으로 처리하는 Spark기반으로 바꿔 빠른 처리 속도를 보여주는 것이 프로젝트의 목표였습니다.
기존에 Hadoop의 Mapreduce로 짜져 있던 선형회귀, Knn알고리즘을 Spark코드로 변경하였습니다. 또 원본 데이터를 기존에 Hive로 가공하였는데 이것 또한 Spark로 가공하는 단계를 거쳤습니다. 앞서 언급한 문제 외에 타부서와의 이해관계문제도 얽히는 등 여러 가지 문제가 있었지만 발로 뛰어다니며 해결하였습니다. 결국, Hadoop기반의 빅 데이터 분석 및 예측 프로젝트를 Spark기반으로 업그레이드하였습니다. 연구소에서 근무하며 빅 데이터에 관련하여 학회의 최신 동향을 알게 되었고, 현업에서 겪을 법한 여러 문제를 경험하며 문제 해결 능력을 기른 뜻 깊은 경험이었습니다.
